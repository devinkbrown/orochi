# Orochi Mesh and S2S Architecture

This document describes the Suimyaku mesh and server-to-server layer as it exists in the current source tree. The planning documents define the direction: Suimyaku is the S2S CRDT mesh, Tsumugi is the secure handshake layer, Sazanami is gossip, and Goryu is the CRDT library ([docs/planning/04-suimyaku-mesh.md:5](../planning/04-suimyaku-mesh.md:5)); the S2S protocol plan explicitly rejects TS6/SID identity in favor of a sovereign node identity ([docs/planning/09-s2s-protocol.md:3](../planning/09-s2s-protocol.md:3), [docs/planning/09-s2s-protocol.md:6](../planning/09-s2s-protocol.md:6)). The code today implements a clean-room Orochi mesh, not a C Ophion clone.

## Layer Map

| Layer | Source | Current role |
| --- | --- | --- |
| IRC daemon integration | `src/daemon/server.zig` | Owns sockets, peer connection lifecycle, local world projection, membership burst, relay delivery, mesh operator commands, partition transition logging, oper grants, and session reclaim. |
| Secured link adapter | `src/daemon/secured_s2s_link.zig` | Frames the Tsumugi TOFU prekey and AKE messages, authenticates the peer identity, then passes raw CRDT/S2S bytes to `S2sLink`. |
| Plain link adapter | `src/daemon/s2s_link.zig` | Buffering adapter around the pure `S2sPeer`; owns per-peer channel CRDT state and outbound byte accumulation. |
| Pure peer driver | `src/substrate/suimyaku/s2s_peer.zig` | Streams length-prefixed frames, performs mesh handshake/burst/delta/gossip dispatch, queues inbound user messages and oper grants, and maintains per-peer registry and route state. |
| Mesh state substrate | `src/substrate/suimyaku/*.zig` | CRDT channel state, bounded membership views, gossip rounds, route tables, server registry, Merkle-guided repair, message relay, and partition analysis. |
| Wire frame codec | `src/proto/s2s_frame.zig` | Five-byte S2S frame header: frame type plus little-endian payload length. |

## Identities And Registries

The current hot path uses `u64` node identifiers ([src/substrate/suimyaku/membership_view.zig:11](../../src/substrate/suimyaku/membership_view.zig:11)). `server_registry.zig` documents that `NodeId` is the sole mesh identity and that there is no legacy TS6 SID concept ([src/substrate/suimyaku/server_registry.zig:6](../../src/substrate/suimyaku/server_registry.zig:6)). Registry entries store `node_id`, display `name`, description, hop count, optional `uplink`, and `last_seen_ms` ([src/substrate/suimyaku/server_registry.zig:42](../../src/substrate/suimyaku/server_registry.zig:42)).

On a raw `S2sLink`, the adapter initializes the channel CRDT replica lane from the local node id ([src/daemon/s2s_link.zig:80](../../src/daemon/s2s_link.zig:80), [src/daemon/s2s_link.zig:82](../../src/daemon/s2s_link.zig:82)). On a secured link, the authenticated Tsumugi peer short id becomes the inner link's remote node id, while the local node id is derived from the configured node identity ([src/daemon/secured_s2s_link.zig:264](../../src/daemon/secured_s2s_link.zig:264), [src/daemon/secured_s2s_link.zig:270](../../src/daemon/secured_s2s_link.zig:270)).

The daemon also keeps operational mesh state outside the substrate: peer health, a bounded mesh event log, and a persistent partition `SeenSet` ([src/daemon/server.zig:1388](../../src/daemon/server.zig:1388), [src/daemon/server.zig:1391](../../src/daemon/server.zig:1391), [src/daemon/server.zig:1398](../../src/daemon/server.zig:1398)).

## S2S Frame Codec

`s2s_frame.zig` defines a simple length-prefixed frame. The header is exactly five bytes: one `u8` frame type tag and a little-endian `u32` payload length, followed by the payload bytes ([src/proto/s2s_frame.zig:1](../../src/proto/s2s_frame.zig:1), [src/proto/s2s_frame.zig:3](../../src/proto/s2s_frame.zig:3)). The default maximum frame size is 1 MiB ([src/proto/s2s_frame.zig:11](../../src/proto/s2s_frame.zig:11)).

| Frame type | Tag | Handled in current peer driver |
| --- | ---: | --- |
| `HANDSHAKE` | `0x01` | Exchanges node id, epoch, server name, and description. |
| `BURST` | `0x02` | Applies a serialized channel CRDT state burst. |
| `DELTA` | `0x03` | Merges a serialized CRDT delta. |
| `GOSSIP` | `0x04` | Applies gossip membership payloads. |
| `PING` / `PONG` | `0x05` / `0x06` | PING replies with matching PONG payload. |
| `QUIT` | `0x07` | Closes the remote peer state. |
| `MEMBERSHIP` | `0x08` | Applies remote channel-member route table facts. |
| `MESSAGE` | `0x09` | Queues cross-node PRIVMSG/NOTICE/TAGMSG payloads for daemon delivery. |
| `OPER_GRANT` | `0x0A` | Queues signed oper-grant payloads for daemon verification. |

The enum and tag mapping are in `FrameType` ([src/proto/s2s_frame.zig:26](../../src/proto/s2s_frame.zig:26)), and the streaming decoder buffers partial input until a complete frame is available ([src/proto/s2s_frame.zig:116](../../src/proto/s2s_frame.zig:116)). Unknown frame tags are rejected as malformed ([src/proto/s2s_frame.zig:123](../../src/proto/s2s_frame.zig:123)).

## Peer Handshake And Link Adapters

The pure peer driver has an internal Suimyaku handshake payload with magic `S2PH`, version `1`, node id, epoch, name, and description ([src/substrate/suimyaku/s2s_peer.zig:29](../../src/substrate/suimyaku/s2s_peer.zig:29), [src/substrate/suimyaku/s2s_peer.zig:89](../../src/substrate/suimyaku/s2s_peer.zig:89), [src/substrate/suimyaku/s2s_peer.zig:494](../../src/substrate/suimyaku/s2s_peer.zig:494)). If the expected remote id is zero, the receiver adopts the id from the first handshake; otherwise it rejects an unexpected id ([src/substrate/suimyaku/s2s_peer.zig:386](../../src/substrate/suimyaku/s2s_peer.zig:386), [src/substrate/suimyaku/s2s_peer.zig:391](../../src/substrate/suimyaku/s2s_peer.zig:391)).

Once established, the peer records the remote server in the registry, maps the remote server name to the remote node in the route table, establishes the link session, clears pending link-session outbound data, and emits a CRDT burst ([src/substrate/suimyaku/s2s_peer.zig:397](../../src/substrate/suimyaku/s2s_peer.zig:397), [src/substrate/suimyaku/s2s_peer.zig:414](../../src/substrate/suimyaku/s2s_peer.zig:414), [src/substrate/suimyaku/s2s_peer.zig:422](../../src/substrate/suimyaku/s2s_peer.zig:422), [src/substrate/suimyaku/s2s_peer.zig:399](../../src/substrate/suimyaku/s2s_peer.zig:399)).

`S2sLink` is reactor-independent: it owns the per-peer CRDT state, a stable monotonic clock cell, and an outbound buffer used by the peer driver's `ByteSink` ([src/daemon/s2s_link.zig:1](../../src/daemon/s2s_link.zig:1), [src/daemon/s2s_link.zig:46](../../src/daemon/s2s_link.zig:46), [src/daemon/s2s_link.zig:62](../../src/daemon/s2s_link.zig:62)). The daemon calls `start` after outbound connect completion for plaintext links ([src/daemon/server.zig:2719](../../src/daemon/server.zig:2719)).

`SecuredLink` wraps the same inner path with Tsumugi. During the handshake only the TOFU prekey preamble and the two AKE messages are length-prefixed with a four-byte little-endian length; after establishment, bytes are passed raw to `S2sLink` because `s2s_frame` already frames the CRDT stream ([src/daemon/secured_s2s_link.zig:1](../../src/daemon/secured_s2s_link.zig:1), [src/daemon/secured_s2s_link.zig:5](../../src/daemon/secured_s2s_link.zig:5), [src/daemon/secured_s2s_link.zig:197](../../src/daemon/secured_s2s_link.zig:197)). The responder immediately queues its signed prekey; the initiator verifies the prekey and optional trust pin before opening the Tsumugi session ([src/daemon/secured_s2s_link.zig:71](../../src/daemon/secured_s2s_link.zig:71), [src/daemon/secured_s2s_link.zig:95](../../src/daemon/secured_s2s_link.zig:95), [src/daemon/secured_s2s_link.zig:231](../../src/daemon/secured_s2s_link.zig:231), [src/daemon/secured_s2s_link.zig:237](../../src/daemon/secured_s2s_link.zig:237)).

The secured adapter exposes `peerNodeKey`, the authenticated raw Ed25519 signing public key recovered from the Tsumugi session, and the daemon uses that key to verify cross-mesh oper grants ([src/daemon/secured_s2s_link.zig:127](../../src/daemon/secured_s2s_link.zig:127), [src/daemon/server.zig:2650](../../src/daemon/server.zig:2650)).

## Channel CRDT Model

The implemented channel CRDT covers one channel. Membership is an add-wins observed-remove set keyed by stable `MemberId`, and modes are last-writer-wins registers ordered by `(HLC, replica_id)` ([src/substrate/suimyaku/channel_crdt.zig:1](../../src/substrate/suimyaku/channel_crdt.zig:1), [src/substrate/suimyaku/channel_crdt.zig:3](../../src/substrate/suimyaku/channel_crdt.zig:3)). Member status stores voice/op/owner/founder bits ([src/substrate/suimyaku/channel_crdt.zig:23](../../src/substrate/suimyaku/channel_crdt.zig:23)).

| Entity | Current representation | Merge behavior |
| --- | --- | --- |
| Membership | Per-member add dots plus causal context (`MemberEntry.adds`, `MemberEntry.context`). | Local join increments the version vector, observes the new dot, and appends a `MemberAdd`; local part clears observed adds; merge removes adds covered by remote context and keeps concurrent live adds. |
| Member status | `MemberStatus` bitset stored on each live add. | `memberStatus` unions the status bits across live adds, so concurrent adds accumulate prefix bits. |
| Boolean channel modes | Goryu LWW registers for invite-only, moderated, no-external, topic-protected, secret. | Register merge decides by timestamp/replica ordering. |
| Parameter modes | LWW registers for key and limit. | Same LWW path as booleans, with bounded key length and `present` flags. |

The join mutator creates a new dot from the version vector, advances HLC, observes the dot in both local and delta contexts, removes covered adds while keeping the new dot, and returns a delta CRDT ([src/substrate/suimyaku/channel_crdt.zig:177](../../src/substrate/suimyaku/channel_crdt.zig:177), [src/substrate/suimyaku/channel_crdt.zig:186](../../src/substrate/suimyaku/channel_crdt.zig:186), [src/substrate/suimyaku/channel_crdt.zig:192](../../src/substrate/suimyaku/channel_crdt.zig:192)). Merge combines HLC, version vectors, member adds/contexts, and mode registers ([src/substrate/suimyaku/channel_crdt.zig:226](../../src/substrate/suimyaku/channel_crdt.zig:226), [src/substrate/suimyaku/channel_crdt.zig:242](../../src/substrate/suimyaku/channel_crdt.zig:242)). The regression tests explicitly cover add-wins rejoin over observed part and netsplit/heal convergence ([src/substrate/suimyaku/channel_crdt.zig:521](../../src/substrate/suimyaku/channel_crdt.zig:521), [src/substrate/suimyaku/channel_crdt.zig:556](../../src/substrate/suimyaku/channel_crdt.zig:556)).

## Burst, Delta, And Anti-Entropy

`burst.zig` serializes a channel CRDT with magic `SBST`, version `1`, a header record, member delta records, and mode records ([src/substrate/suimyaku/burst.zig:39](../../src/substrate/suimyaku/burst.zig:39), [src/substrate/suimyaku/burst.zig:45](../../src/substrate/suimyaku/burst.zig:45)). The header carries HLC and the version vector ([src/substrate/suimyaku/burst.zig:103](../../src/substrate/suimyaku/burst.zig:103)); member records use `delta_codec` membership views with context dots, live add dots, and derived remove dots ([src/substrate/suimyaku/burst.zig:130](../../src/substrate/suimyaku/burst.zig:130), [src/substrate/suimyaku/burst.zig:147](../../src/substrate/suimyaku/burst.zig:147), [src/substrate/suimyaku/burst.zig:214](../../src/substrate/suimyaku/burst.zig:214)). Applying a burst builds an incoming CRDT and merges it into the target ([src/substrate/suimyaku/burst.zig:72](../../src/substrate/suimyaku/burst.zig:72), [src/substrate/suimyaku/burst.zig:95](../../src/substrate/suimyaku/burst.zig:95)).

`LinkSession` is the pure orchestration layer for initial burst, live deltas, gossip payloads, and Merkle-guided repair ([src/substrate/suimyaku/link_session.zig:1](../../src/substrate/suimyaku/link_session.zig:1), [src/substrate/suimyaku/link_session.zig:3](../../src/substrate/suimyaku/link_session.zig:3)). On establishment it observes the remote node in gossip and enqueues a burst ([src/substrate/suimyaku/link_session.zig:123](../../src/substrate/suimyaku/link_session.zig:123)). On tick it emits gossip after `gossip_interval_ms` and repair summaries after `repair_interval_ms` ([src/substrate/suimyaku/link_session.zig:132](../../src/substrate/suimyaku/link_session.zig:132), [src/substrate/suimyaku/link_session.zig:139](../../src/substrate/suimyaku/link_session.zig:139), [src/substrate/suimyaku/link_session.zig:156](../../src/substrate/suimyaku/link_session.zig:156)).

The repair module summarizes each member and present mode into Merkle entries, diffs local and remote summaries, builds key requests, builds record responses, and applies responses through a temporary CRDT merge ([src/substrate/suimyaku/anti_entropy_repair.zig:83](../../src/substrate/suimyaku/anti_entropy_repair.zig:83), [src/substrate/suimyaku/anti_entropy_repair.zig:109](../../src/substrate/suimyaku/anti_entropy_repair.zig:109), [src/substrate/suimyaku/anti_entropy_repair.zig:133](../../src/substrate/suimyaku/anti_entropy_repair.zig:133), [src/substrate/suimyaku/anti_entropy_repair.zig:143](../../src/substrate/suimyaku/anti_entropy_repair.zig:143), [src/substrate/suimyaku/anti_entropy_repair.zig:156](../../src/substrate/suimyaku/anti_entropy_repair.zig:156)). `S2sPeer.repairRoot` exposes the current Merkle root for its state ([src/substrate/suimyaku/s2s_peer.zig:278](../../src/substrate/suimyaku/s2s_peer.zig:278)).

Important current-code boundary: `s2s_peer.zig` directly dispatches S2S `BURST`, `DELTA`, and `GOSSIP` frames, while `LinkSession` has its own repair-summary/request/response message-kind path over `peer_link` payloads ([src/substrate/suimyaku/s2s_peer.zig:284](../../src/substrate/suimyaku/s2s_peer.zig:284), [src/substrate/suimyaku/link_session.zig:200](../../src/substrate/suimyaku/link_session.zig:200)). The daemon's live S2S path uses the `S2sPeer` frame path through `S2sLink`/`SecuredLink`.

## Gossip And Membership View

`membership_view.zig` implements a bounded HyParView-shaped partial view: a small active view and larger passive view, with caller-supplied time and deterministic randomness ([src/substrate/suimyaku/membership_view.zig:1](../../src/substrate/suimyaku/membership_view.zig:1), [src/substrate/suimyaku/membership_view.zig:40](../../src/substrate/suimyaku/membership_view.zig:40)). Joining an overflowed active view demotes one active peer to passive ([src/substrate/suimyaku/membership_view.zig:169](../../src/substrate/suimyaku/membership_view.zig:169), [src/substrate/suimyaku/membership_view.zig:195](../../src/substrate/suimyaku/membership_view.zig:195)). Active failure removes the failed active member and may promote a passive candidate ([src/substrate/suimyaku/membership_view.zig:227](../../src/substrate/suimyaku/membership_view.zig:227), [src/substrate/suimyaku/membership_view.zig:247](../../src/substrate/suimyaku/membership_view.zig:247)).

`gossip_round.zig` combines the bounded view with Sazanami liveness state. Members can be `alive`, `suspect`, `dead`, or `left` ([src/substrate/suimyaku/gossip_round.zig:8](../../src/substrate/suimyaku/gossip_round.zig:8)). Suspicion records witnesses and transitions to dead only after the configured witness quorum and suspicion timeout ([src/substrate/suimyaku/gossip_round.zig:111](../../src/substrate/suimyaku/gossip_round.zig:111), [src/substrate/suimyaku/gossip_round.zig:183](../../src/substrate/suimyaku/gossip_round.zig:183)). A gossip round applies received payloads, reaps suspects, selects peers by fanout, and builds a payload of member deltas and suspicions ([src/substrate/suimyaku/gossip_round.zig:308](../../src/substrate/suimyaku/gossip_round.zig:308), [src/substrate/suimyaku/gossip_round.zig:320](../../src/substrate/suimyaku/gossip_round.zig:320), [src/substrate/suimyaku/gossip_round.zig:331](../../src/substrate/suimyaku/gossip_round.zig:331), [src/substrate/suimyaku/gossip_round.zig:387](../../src/substrate/suimyaku/gossip_round.zig:387)).

The peer driver serializes gossip payloads as little-endian origin/incarnation, varint member-delta count, per-delta id/state/incarnation, varint suspicion count, and per-suspicion node/incarnation/witness ([src/substrate/suimyaku/s2s_peer.zig:523](../../src/substrate/suimyaku/s2s_peer.zig:523), [src/substrate/suimyaku/s2s_peer.zig:543](../../src/substrate/suimyaku/s2s_peer.zig:543)).

## Route Table And Remote Membership Projection

The route table has two separate projections: nick-to-node routing and per-channel remote member rosters. Channel routing tracks node fanout counts, while `channel_members` stores remote `nick`, `node`, `status`, and `hlc` for NAMES/WHO-style projection ([src/substrate/suimyaku/route_table.zig:141](../../src/substrate/suimyaku/route_table.zig:141), [src/substrate/suimyaku/route_table.zig:148](../../src/substrate/suimyaku/route_table.zig:148), [src/substrate/suimyaku/route_table.zig:113](../../src/substrate/suimyaku/route_table.zig:113)).

`MEMBERSHIP` frame payloads are compact binary records: `present`, `status`, `origin_node`, `hlc`, channel length/string, and nick length/string ([src/proto/membership_event.zig:1](../../src/proto/membership_event.zig:1), [src/proto/membership_event.zig:9](../../src/proto/membership_event.zig:9)). The peer driver decodes inbound membership and applies it to the route table; malformed payloads are dropped without faulting the link ([src/substrate/suimyaku/s2s_peer.zig:350](../../src/substrate/suimyaku/s2s_peer.zig:350)). Route-table membership updates are LWW by HLC; stale updates are ignored, newer present updates replace status/node, and newer parts remove the entry and prune empty channels ([src/substrate/suimyaku/route_table.zig:275](../../src/substrate/suimyaku/route_table.zig:275), [src/substrate/suimyaku/route_table.zig:291](../../src/substrate/suimyaku/route_table.zig:291)).

The daemon announces local membership changes to all established peer links. JOIN sends the member's current mode bits after local join handling, and PART sends a `present=false` event after removing the local membership ([src/daemon/server.zig:3991](../../src/daemon/server.zig:3991), [src/daemon/server.zig:3994](../../src/daemon/server.zig:3994), [src/daemon/server.zig:4042](../../src/daemon/server.zig:4042), [src/daemon/server.zig:4045](../../src/daemon/server.zig:4045)). The shared announcer sends over both secured and plaintext established links and flushes each link's outbound buffer ([src/daemon/server.zig:3640](../../src/daemon/server.zig:3640), [src/daemon/server.zig:3644](../../src/daemon/server.zig:3644)).

On link-up, the daemon sends a one-time full local channel-member roster to that peer so the peer does not have to wait for future joins to build a roster ([src/daemon/server.zig:2663](../../src/daemon/server.zig:2663), [src/daemon/server.zig:2699](../../src/daemon/server.zig:2699), [src/daemon/server.zig:3611](../../src/daemon/server.zig:3611)). The burst walks all local channels and local client members, sends their status bits, then flushes the peer link ([src/daemon/server.zig:3614](../../src/daemon/server.zig:3614), [src/daemon/server.zig:3620](../../src/daemon/server.zig:3620), [src/daemon/server.zig:3631](../../src/daemon/server.zig:3631)).

## Cross-Node Message Relay

`message_relay.zig` defines the cross-node user-message schema for relayed PRIVMSG, NOTICE, and TAGMSG. It is a canonical CoilPack map with stable field order, carrying verb, target, source nick/prefix, account, tags, text, origin node, and HLC ([src/substrate/suimyaku/message_relay.zig:1](../../src/substrate/suimyaku/message_relay.zig:1), [src/substrate/suimyaku/message_relay.zig:11](../../src/substrate/suimyaku/message_relay.zig:11), [src/substrate/suimyaku/message_relay.zig:51](../../src/substrate/suimyaku/message_relay.zig:51)). Each peer has a bounded `SeenSet` keyed by `(origin_node, hlc)` to suppress duplicates and echoes ([src/substrate/suimyaku/message_relay.zig:144](../../src/substrate/suimyaku/message_relay.zig:144)).

Inbound `MESSAGE` frames are decoded, loop-checked, and queued for daemon delivery; malformed payloads are ignored ([src/substrate/suimyaku/s2s_peer.zig:320](../../src/substrate/suimyaku/s2s_peer.zig:320)). Outbound messages are observed in the same seen set before being encoded and emitted as `MESSAGE` frames ([src/substrate/suimyaku/s2s_peer.zig:335](../../src/substrate/suimyaku/s2s_peer.zig:335)).

The daemon drains queued inbound messages from both link types after driving the link and calls `deliverRelay` ([src/daemon/server.zig:2642](../../src/daemon/server.zig:2642), [src/daemon/server.zig:2691](../../src/daemon/server.zig:2691)). `deliverRelay` delivers to local channel members or a local nick, then re-forwards for multi-hop using a route-table-derived scope; unknown nick routes widen to all peers ([src/daemon/server.zig:3106](../../src/daemon/server.zig:3106), [src/daemon/server.zig:3112](../../src/daemon/server.zig:3112)).

Outbound channel messages are relayed only to peers that have announced members of the channel, and outbound nick messages are first sent to the peer that claims the nick route; if no nick route is known, the daemon floods all established peers ([src/daemon/server.zig:11050](../../src/daemon/server.zig:11050), [src/daemon/server.zig:11058](../../src/daemon/server.zig:11058), [src/daemon/server.zig:11077](../../src/daemon/server.zig:11077), [src/daemon/server.zig:11096](../../src/daemon/server.zig:11096)). The peer selection function supports `.all`, `.channel`, and `.nick` scopes and checks channel roster or nick route availability on each link ([src/daemon/server.zig:3017](../../src/daemon/server.zig:3017), [src/daemon/server.zig:3026](../../src/daemon/server.zig:3026), [src/daemon/server.zig:3052](../../src/daemon/server.zig:3052)).

## Netsplit And Netjoin Batches

`netsplit_batch.zig` is a pure formatter for IRCv3 netsplit/netjoin `BATCH` messages. It allocates nothing and leaves recipient selection to the daemon ([src/daemon/netsplit_batch.zig:1](../../src/daemon/netsplit_batch.zig:1)). `writeNetsplit` wraps ghost QUIT lines in a `netsplit` batch; `writeNetjoin` wraps ghost JOIN lines in a `netjoin` batch ([src/daemon/netsplit_batch.zig:27](../../src/daemon/netsplit_batch.zig:27), [src/daemon/netsplit_batch.zig:54](../../src/daemon/netsplit_batch.zig:54)).

The current daemon calls the netsplit path on S2S peer drop, before deinitializing the link, so it can still read the peer's remote member roster ([src/daemon/server.zig:3156](../../src/daemon/server.zig:3156), [src/daemon/server.zig:3162](../../src/daemon/server.zig:3162), [src/daemon/server.zig:3177](../../src/daemon/server.zig:3177)). For each local channel, it reads the dropped peer's remote members, renders ghost prefixes as `nick!*@remote_server`, emits a netsplit batch, and sends it to local channel members ([src/daemon/server.zig:3571](../../src/daemon/server.zig:3571), [src/daemon/server.zig:3575](../../src/daemon/server.zig:3575), [src/daemon/server.zig:3583](../../src/daemon/server.zig:3583), [src/daemon/server.zig:3594](../../src/daemon/server.zig:3594), [src/daemon/server.zig:3599](../../src/daemon/server.zig:3599)).

Current-code boundary: the netjoin formatter exists and is tested, but there is no daemon call site for `writeNetjoin` in `server.zig` in the current source. Link-up currently uses the `MEMBERSHIP` roster burst described above rather than rendering netjoin batches to local clients.

## Multi-Hop Partition Detection And Quorum

`partition_detector.zig` is pure graph logic. Callers provide known nodes, liveness, and undirected edges; dead nodes are excluded from counts, reachability, components, and quorum, while suspect nodes remain in the graph ([src/substrate/suimyaku/partition_detector.zig:1](../../src/substrate/suimyaku/partition_detector.zig:1), [src/substrate/suimyaku/partition_detector.zig:10](../../src/substrate/suimyaku/partition_detector.zig:10)). The detector reports reachability, partition status, connected-component count, and strict-majority quorum from the local node's component ([src/substrate/suimyaku/partition_detector.zig:144](../../src/substrate/suimyaku/partition_detector.zig:144), [src/substrate/suimyaku/partition_detector.zig:189](../../src/substrate/suimyaku/partition_detector.zig:189), [src/substrate/suimyaku/partition_detector.zig:200](../../src/substrate/suimyaku/partition_detector.zig:200), [src/substrate/suimyaku/partition_detector.zig:213](../../src/substrate/suimyaku/partition_detector.zig:213)).

For daemon snapshots, `TopoNode` stores `node_id` plus optional `uplink`, and `analyze` treats each uplink as an undirected edge while always including the local node ([src/substrate/suimyaku/partition_detector.zig:383](../../src/substrate/suimyaku/partition_detector.zig:383), [src/substrate/suimyaku/partition_detector.zig:404](../../src/substrate/suimyaku/partition_detector.zig:404)). `SeenSet` remembers observed node ids with last-seen time so a node that vanishes with a dropped link remains expected until TTL expiry ([src/substrate/suimyaku/partition_detector.zig:461](../../src/substrate/suimyaku/partition_detector.zig:461), [src/substrate/suimyaku/partition_detector.zig:480](../../src/substrate/suimyaku/partition_detector.zig:480), [src/substrate/suimyaku/partition_detector.zig:493](../../src/substrate/suimyaku/partition_detector.zig:493)).

The daemon assembles topology from every established direct peer plus each peer's gossiped registry view. For plaintext links it uses `remoteNodeId`; for secured links it uses the authenticated peer short id; both then append `collectTopology` from the link ([src/daemon/server.zig:10203](../../src/daemon/server.zig:10203), [src/daemon/server.zig:10213](../../src/daemon/server.zig:10213), [src/daemon/server.zig:10221](../../src/daemon/server.zig:10221), [src/daemon/server.zig:10222](../../src/daemon/server.zig:10222), [src/daemon/server.zig:10230](../../src/daemon/server.zig:10230)).

`updatePartitionTransitions` runs on link establishment and drop. It observes local and visible topology nodes into the daemon's persistent `SeenSet`, prunes nodes older than five minutes, builds edges from the current topology, and logs a `.split` or `.heal` mesh event only when the partitioned boolean crosses state ([src/daemon/server.zig:2635](../../src/daemon/server.zig:2635), [src/daemon/server.zig:2680](../../src/daemon/server.zig:2680), [src/daemon/server.zig:3166](../../src/daemon/server.zig:3166), [src/daemon/server.zig:10201](../../src/daemon/server.zig:10201), [src/daemon/server.zig:10236](../../src/daemon/server.zig:10236), [src/daemon/server.zig:10250](../../src/daemon/server.zig:10250), [src/daemon/server.zig:10276](../../src/daemon/server.zig:10276)).

`MESH` renders reachability and partition counts from `partition_detector.analyze`, then reports quorum as `reachable_nodes > partitioned_nodes`, equivalent to local reachability being more than half of known nodes ([src/daemon/server.zig:10355](../../src/daemon/server.zig:10355), [src/daemon/server.zig:10363](../../src/daemon/server.zig:10363), [src/daemon/server.zig:10389](../../src/daemon/server.zig:10389)).

## Secure Cross-Mesh Oper Grants

Cross-mesh oper grants never carry the operator password. `oper_cred_share.zig` defines a signed, expiring authorization grant with account, privilege bitfield, class, title, issuer node, incarnation, issue time, and expiry ([src/proto/oper_cred_share.zig:1](../../src/proto/oper_cred_share.zig:1), [src/proto/oper_cred_share.zig:41](../../src/proto/oper_cred_share.zig:41)). The signed region is fixed-order and length-prefixed; Ed25519 signs exactly those canonical bytes, and verification checks structure, signature, and expiry ([src/proto/oper_cred_share.zig:163](../../src/proto/oper_cred_share.zig:163), [src/proto/oper_cred_share.zig:213](../../src/proto/oper_cred_share.zig:213), [src/proto/oper_cred_share.zig:228](../../src/proto/oper_cred_share.zig:228)). The fixed-capacity registry merges grants by account and rejects older-or-equal incarnations/replays ([src/proto/oper_cred_share.zig:325](../../src/proto/oper_cred_share.zig:325), [src/proto/oper_cred_share.zig:372](../../src/proto/oper_cred_share.zig:372)).

The daemon stores recognized grants in `oper_grants` ([src/daemon/server.zig:1433](../../src/daemon/server.zig:1433)). Locally authorized SASL operators mint a 24-hour grant signed by this node's Ed25519 key and broadcast it to established secured peers only ([src/daemon/server.zig:8285](../../src/daemon/server.zig:8285), [src/daemon/server.zig:8290](../../src/daemon/server.zig:8290), [src/daemon/server.zig:8341](../../src/daemon/server.zig:8341), [src/daemon/server.zig:8361](../../src/daemon/server.zig:8361), [src/daemon/server.zig:8367](../../src/daemon/server.zig:8367)). The plaintext S2S path is deliberately skipped for grant broadcast because it lacks an authenticated peer identity ([src/daemon/server.zig:8367](../../src/daemon/server.zig:8367)).

Inbound `OPER_GRANT` frames are opaque to the link layer; secured `driveS2sSecured` drains them and verifies each against `peerNodeKey` before applying it to local authority ([src/substrate/suimyaku/s2s_peer.zig:302](../../src/substrate/suimyaku/s2s_peer.zig:302), [src/daemon/server.zig:2650](../../src/daemon/server.zig:2650)). Accepted grants are logged and immediately applied to already-connected sessions for the same account ([src/daemon/server.zig:8304](../../src/daemon/server.zig:8304), [src/daemon/server.zig:8311](../../src/daemon/server.zig:8311), [src/daemon/server.zig:8313](../../src/daemon/server.zig:8313), [src/daemon/server.zig:8322](../../src/daemon/server.zig:8322)). If an account has no local oper binding, `elevateOperFromAccount` can still derive operator privileges from a live cross-mesh grant ([src/daemon/server.zig:8385](../../src/daemon/server.zig:8385), [src/daemon/server.zig:8399](../../src/daemon/server.zig:8399)).

Operators can inspect recognized grants with `MESH GRANTS`, which lists account, class, title, issuer, and remaining TTL ([src/daemon/server.zig:10298](../../src/daemon/server.zig:10298), [src/daemon/server.zig:10301](../../src/daemon/server.zig:10301)).

## Cross-Server Session Reclaim

`session_reclaim_mesh.zig` defines portable reclaim tokens for session resume across the mesh. The token is canonical length-prefixed fields plus an HMAC-SHA256 tag under the configured mesh shared key ([src/proto/session_reclaim_mesh.zig:1](../../src/proto/session_reclaim_mesh.zig:1), [src/proto/session_reclaim_mesh.zig:8](../../src/proto/session_reclaim_mesh.zig:8)). Fields include account, session id, origin node, issue time, expiry time, and a replay nonce ([src/proto/session_reclaim_mesh.zig:36](../../src/proto/session_reclaim_mesh.zig:36)). `open` validates the magic, parses the body, verifies the tag in constant time, and rejects expired tokens ([src/proto/session_reclaim_mesh.zig:170](../../src/proto/session_reclaim_mesh.zig:170), [src/proto/session_reclaim_mesh.zig:194](../../src/proto/session_reclaim_mesh.zig:194), [src/proto/session_reclaim_mesh.zig:200](../../src/proto/session_reclaim_mesh.zig:200)).

The decision function is separate from verification. After a token is open, it denies expired or replayed tokens, grants local reclaim if this node holds the session, denies unknown if the origin is this node but the session is gone, or redirects to the origin node otherwise ([src/proto/session_reclaim_mesh.zig:212](../../src/proto/session_reclaim_mesh.zig:212), [src/proto/session_reclaim_mesh.zig:222](../../src/proto/session_reclaim_mesh.zig:222)). A bounded `ReplayRing` records nonces and reports repeats ([src/proto/session_reclaim_mesh.zig:236](../../src/proto/session_reclaim_mesh.zig:236), [src/proto/session_reclaim_mesh.zig:251](../../src/proto/session_reclaim_mesh.zig:251)).

In the daemon, a `SESSION RESUME` token with non-local length is treated as a mesh-sealed token and sent to `handleMeshReclaim` ([src/daemon/server.zig:8903](../../src/daemon/server.zig:8903), [src/daemon/server.zig:8914](../../src/daemon/server.zig:8914)). `handleMeshReclaim` requires `mesh_pass`, hex-decodes the token, opens it, checks that the token account equals the caller's authenticated account, checks nonce replay, then follows the decision result by reclaiming a local ghost, returning a redirect, or returning a structured failure ([src/daemon/server.zig:8939](../../src/daemon/server.zig:8939), [src/daemon/server.zig:8945](../../src/daemon/server.zig:8945), [src/daemon/server.zig:8954](../../src/daemon/server.zig:8954), [src/daemon/server.zig:8960](../../src/daemon/server.zig:8960), [src/daemon/server.zig:8969](../../src/daemon/server.zig:8969), [src/daemon/server.zig:8984](../../src/daemon/server.zig:8984), [src/daemon/server.zig:8987](../../src/daemon/server.zig:8987)).

## Current Implementation Boundaries

The source currently implements a frame-based peer driver and daemon relay path, not the full future banded protocol from the planning documents. `docs/planning/09-s2s-protocol.md` describes future bands, signed deltas, MeshPass, and Plumtree/RBSR work ([docs/planning/09-s2s-protocol.md:120](../planning/09-s2s-protocol.md:120), [docs/planning/09-s2s-protocol.md:214](../planning/09-s2s-protocol.md:214)). The source today has Tsumugi-secured link establishment, CRDT burst/delta/gossip frames, route-table-scoped message relay, membership frame projection, oper-grant sharing over secured links, and mesh-session reclaim. Where the code only provides pure helpers, this document calls that out: `writeNetjoin` exists, but the daemon does not call it; `LinkSession` contains repair message scheduling, but the live daemon peer path dispatches `s2s_frame` messages through `S2sPeer`.
