# Persistence

*Configure OroStore, the Zig-native key/value store that backs accounts and services.*

Onyx Server persistence is OroStore, a Zig-native store rather than LMDB. OroStore is an in-memory typed key/value database backed by a checksummed append-only WAL, snapshot compaction, and a bounded recent-mutation feed (`src/daemon/store.zig:1`, `src/daemon/store.zig:3`).

## Opening a store

The daemon opens the account store when `[sasl].account_db` is set (`src/main.zig:177`). The path is interpreted relative to the daemon's current working directory, because `main.zig` opens it under `std.Io.Dir.cwd()` (`src/main.zig:179`).

```toml
[sasl]
account_db = "/var/lib/orochi/accounts.wal"
```

The account store backs services and SASL verification. Once it is open, the daemon attaches SCRAM and certfp mirrors, wires PLAIN, SCRAM-SHA-256, and EXTERNAL into the server, and exposes account services to command handlers (`src/main.zig:181`, `src/main.zig:182`, `src/main.zig:183`, `src/main.zig:189`, `src/main.zig:192`).

## File layout

`OroStore.open` takes a WAL path and replays `<wal_path>.snap` first, then the WAL (`src/daemon/store.zig:107`, `src/daemon/store.zig:143`). For `account_db = "/var/lib/orochi/accounts.wal"`, the files are:

| File | Purpose |
|---|---|
| `/var/lib/orochi/accounts.wal` | Append-only mutation log. Created if missing (`src/daemon/store.zig:223`). |
| `/var/lib/orochi/accounts.wal.snap` | Snapshot generated by compaction (`src/daemon/store.zig:125`, `src/daemon/store.zig:179`). |

Each appended record is synced after the write (`src/daemon/store.zig:291`, `src/daemon/store.zig:301`). `snapshotAndTruncate` writes the current state to an atomic snapshot, syncs it, replaces the old snapshot, then truncates and syncs the WAL (`src/daemon/store.zig:179`, `src/daemon/store.zig:202`, `src/daemon/store.zig:206`).

## Families

OroStore has typed column families (`src/daemon/store.zig:52`):

| Family | Use |
|---|---|
| `accounts` | Account records. |
| `nicks` | Registered nick data. |
| `chanregs` | Registered channel data. |
| `bans` | Service ban records. |
| `memos` | Memo records. |
| `vhosts` | Virtual host records. |
| `props` | Service/property metadata. |
| `history` | Persistent history records. |

The store also exposes a bounded recent-mutation changefeed carrying a sequence number, family, mutation kind, key, and value (`src/daemon/store.zig:68`, `src/daemon/store.zig:213`).

## Limits

OroStore defaults are a 16 MiB maximum record payload, a 256 MiB maximum WAL replay size, and 64 changefeed entries (`src/daemon/store.zig:11`, `src/daemon/store.zig:12`, `src/daemon/store.zig:13`). A `store.Config.applyToml` helper exists for `[storage]`, but `[storage]` is not part of the daemon config parsed by `src/daemon/config_format.zig` today (`src/daemon/store.zig:26`, `src/daemon/store.zig:37`).
