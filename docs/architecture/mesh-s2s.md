# Orochi mesh and S2S architecture

*The Suimyaku mesh and server-to-server layer as implemented in the current source tree.*

The mesh vocabulary (see the [glossary](../reference/glossary.md)): Suimyaku is the S2S CRDT mesh state model, Tsumugi is the secure handshake layer, Sazanami is the witnessed failure-detection gossip, and Goryu is the CRDT library. The design uses a sovereign node identity and does not key the mesh on server names, origins, or nicks. The code today implements a clean-room Orochi mesh.

## Layer map

| Layer | Source | Current role |
| --- | --- | --- |
| IRC daemon integration | `src/daemon/server.zig` | Owns sockets, peer connection lifecycle, local world projection, membership burst, relay delivery, mesh operator commands, partition transition logging, oper grants, and session reclaim. |
| Secured link adapter | `src/daemon/secured_s2s_link.zig` | Frames the Tsumugi TOFU prekey and AKE messages, authenticates the peer identity, then passes decrypted inner CRDT/S2S bytes to `S2sLink`. |
| Plain link adapter | `src/daemon/s2s_link.zig` | Buffering adapter around the pure `S2sPeer`; owns per-peer channel CRDT state and outbound byte accumulation. |
| Pure peer driver | `src/substrate/suimyaku/s2s_peer.zig` | Streams length-prefixed frames, performs mesh handshake/burst/delta/gossip dispatch, queues inbound user messages and oper grants, and maintains per-peer registry and route state. |
| Mesh state substrate | `src/substrate/suimyaku/*.zig` | CRDT channel state, bounded membership views, gossip rounds, route tables, server registry, Merkle-guided repair, message relay, and partition analysis. |
| Wire frame codec | `src/proto/s2s_frame.zig` | Five-byte S2S frame header: frame type plus little-endian payload length. |

## Identities and registries

The secured hot path authenticates a canonical 20-byte Tsumugi node id and derives the Suimyaku `u64` routing handle from it with `node_short_id.shortId`; that `shortId` is the mesh handle used by `S2sPeer`, route tables, and gossip, while server names, origins, and nicks are not identities ([src/crypto/node_short_id.zig:4](../../src/crypto/node_short_id.zig#L4), [src/crypto/node_short_id.zig:12](../../src/crypto/node_short_id.zig#L12), [src/crypto/tsumugi_session.zig:13](../../src/crypto/tsumugi_session.zig#L13), [src/crypto/tsumugi_session.zig:143](../../src/crypto/tsumugi_session.zig#L143)). Registry entries store `node_id`, display `name`, description, hop count, optional `uplink`, and `last_seen_ms` ([src/substrate/suimyaku/server_registry.zig:42](../../src/substrate/suimyaku/server_registry.zig#L42)).

On a raw `S2sLink`, the adapter initializes the channel CRDT replica lane from the local node id ([src/daemon/s2s_link.zig:103](../../src/daemon/s2s_link.zig#L103), [src/daemon/s2s_link.zig:105](../../src/daemon/s2s_link.zig#L105)). On a secured link, the authenticated Tsumugi peer short id becomes the inner link's remote node id, while the local node id is derived from the configured node identity ([src/daemon/secured_s2s_link.zig:845](../../src/daemon/secured_s2s_link.zig#L845), [src/daemon/secured_s2s_link.zig:850](../../src/daemon/secured_s2s_link.zig#L850)).

The daemon also keeps operational mesh state outside the substrate: peer health, a bounded mesh event log, and a persistent partition `SeenSet` ([src/daemon/server.zig:1388](../../src/daemon/server.zig#L1388), [src/daemon/server.zig:1391](../../src/daemon/server.zig#L1391), [src/daemon/server.zig:1398](../../src/daemon/server.zig#L1398)).

## S2S frame codec

`s2s_frame.zig` defines a simple length-prefixed frame. The header is exactly five bytes: one `u8` frame type tag and a little-endian `u32` payload length, followed by the payload bytes ([src/proto/s2s_frame.zig:1](../../src/proto/s2s_frame.zig#L1), [src/proto/s2s_frame.zig:3](../../src/proto/s2s_frame.zig#L3)). The default maximum frame size is 1 MiB ([src/proto/s2s_frame.zig:11](../../src/proto/s2s_frame.zig#L11)).

`src/proto/s2s_frame.zig` is now the source of truth for the public S2S frame catalog and the handshake capability catalog. `frame_catalog` records the tag, family, authentication mode, capability gate, and summary for every known frame; the codec tests assert that every `FrameType` is covered exactly once. `capability_catalog` records the stable handshake bits used by the peer driver and Helix resume snapshots.

| `member-oper-info` | `2` | `0x04` | Secured-only real-host and certificate fingerprint propagation for oper-visible identity. |

> Unknown frame tags are forward-tolerant as long as the length header is valid:
> the decoder skips unknown length-delimited frames instead of tearing the link
> down. Emitting a new frame still needs an upgrade or handshake-capability gate;
> otherwise the receiver will discard behavior it does not understand.

> Origin authentication, the per-link vs multi-hop signing modes, and the
> handshake-negotiated signing capability are documented in
> [mesh-security.md](mesh-security.md).

The enum and tag mapping are in `FrameType` ([src/proto/s2s_frame.zig:29](../../src/proto/s2s_frame.zig#L29)), and the streaming decoder buffers partial input until a complete frame is available ([src/proto/s2s_frame.zig:374](../../src/proto/s2s_frame.zig#L374)). Unknown frame tags are skipped after the bounded payload length is validated ([src/proto/s2s_frame.zig:393](../../src/proto/s2s_frame.zig#L393)).

## Peer handshake and link adapters

The pure peer driver has an internal Suimyaku handshake payload with magic `S2PH`, version `2`, node id, epoch, name, description, and a single-byte capability bitfield. v1 peers omit the byte and decode as `caps == 0`; unknown future capability bits are ignored. If the expected remote id is zero, the receiver adopts the id from the first handshake; otherwise it rejects an unexpected id.

Once established, the peer records the remote server in the registry, maps the remote server name to the remote node in the route table, establishes the link session, clears pending link-session outbound data, and emits a CRDT burst ([src/substrate/suimyaku/s2s_peer.zig:397](../../src/substrate/suimyaku/s2s_peer.zig#L397), [src/substrate/suimyaku/s2s_peer.zig:414](../../src/substrate/suimyaku/s2s_peer.zig#L414), [src/substrate/suimyaku/s2s_peer.zig:422](../../src/substrate/suimyaku/s2s_peer.zig#L422), [src/substrate/suimyaku/s2s_peer.zig:399](../../src/substrate/suimyaku/s2s_peer.zig#L399)).

`S2sLink` is reactor-independent: it owns the per-peer CRDT state, a stable monotonic clock cell, and an outbound buffer used by the peer driver's `ByteSink` ([src/daemon/s2s_link.zig:1](../../src/daemon/s2s_link.zig#L1), [src/daemon/s2s_link.zig:46](../../src/daemon/s2s_link.zig#L46), [src/daemon/s2s_link.zig:62](../../src/daemon/s2s_link.zig#L62)). The daemon calls `start` after outbound connect completion for plaintext links ([src/daemon/server.zig:2719](../../src/daemon/server.zig#L2719)).

`SecuredLink` wraps the same inner path with Tsumugi. During the handshake only the TOFU prekey preamble and the two AKE messages are length-prefixed with a four-byte little-endian length ([src/daemon/secured_s2s_link.zig:4](../../src/daemon/secured_s2s_link.zig#L4), [src/daemon/secured_s2s_link.zig:6](../../src/daemon/secured_s2s_link.zig#L6)). Tsumugi is the PQ-hybrid Noise-IK-shaped AKE: Ed25519 is the static node identity, X-Wing transport prekeys provide hybrid KEM entropy, and the initiator node id plus MeshPass bytes are only present inside encrypted M1 ([src/crypto/tsumugi_handshake.zig:4](../../src/crypto/tsumugi_handshake.zig#L4), [src/crypto/tsumugi_handshake.zig:6](../../src/crypto/tsumugi_handshake.zig#L6), [src/crypto/tsumugi_handshake.zig:9](../../src/crypto/tsumugi_handshake.zig#L9), [src/crypto/tsumugi_handshake.zig:370](../../src/crypto/tsumugi_handshake.zig#L370), [src/crypto/tsumugi_handshake.zig:383](../../src/crypto/tsumugi_handshake.zig#L383)). After establishment the post-handshake CRDT byte stream is **encrypted**: `feedInner`/`drainInner` wrap each inner chunk in a ChaCha20-Poly1305 AEAD record keyed on the Tsumugi `Established` send/recv keys with a per-record counter nonce bound as AAD, so a tampered or reordered record drops the link rather than reaching `S2sLink` ([src/daemon/secured_s2s_link.zig:13](../../src/daemon/secured_s2s_link.zig#L13), [src/daemon/secured_s2s_link.zig:45](../../src/daemon/secured_s2s_link.zig#L45), [src/crypto/tsumugi_handshake.zig:183](../../src/crypto/tsumugi_handshake.zig#L183), [src/crypto/tsumugi_handshake.zig:198](../../src/crypto/tsumugi_handshake.zig#L198)). Derived session keys and temporary key material are secure-zeroed on deinit and after use ([src/crypto/tsumugi_handshake.zig:104](../../src/crypto/tsumugi_handshake.zig#L104), [src/crypto/tsumugi_handshake.zig:190](../../src/crypto/tsumugi_handshake.zig#L190), [src/crypto/tsumugi_handshake.zig:609](../../src/crypto/tsumugi_handshake.zig#L609), [src/crypto/tsumugi_handshake.zig:810](../../src/crypto/tsumugi_handshake.zig#L810)). The responder immediately queues its signed prekey; the initiator verifies the prekey and optional trust pin before opening the Tsumugi session ([src/daemon/secured_s2s_link.zig:22](../../src/daemon/secured_s2s_link.zig#L22), [src/daemon/secured_s2s_link.zig:796](../../src/daemon/secured_s2s_link.zig#L796), [src/daemon/secured_s2s_link.zig:833](../../src/daemon/secured_s2s_link.zig#L833)). The end-to-end origin-authentication and capability-negotiation model layered on top of this is documented separately in [mesh-security.md](mesh-security.md).

MeshPass admission is inside that encrypted M1. With signer roots configured, the responder decodes a signed MeshPass token, binds it to the authenticated peer node key, requires the relay role plus control/sync/tsumugi frame families, and carries admitted frame-family rights into the established session; without signer roots it keeps the shared-secret gate and compares the M1 MeshPass bytes in constant time ([src/crypto/tsumugi_handshake.zig:438](../../src/crypto/tsumugi_handshake.zig#L438), [src/crypto/tsumugi_handshake.zig:442](../../src/crypto/tsumugi_handshake.zig#L442), [src/crypto/tsumugi_handshake.zig:445](../../src/crypto/tsumugi_handshake.zig#L445), [src/crypto/tsumugi_handshake.zig:821](../../src/crypto/tsumugi_handshake.zig#L821), [src/crypto/tsumugi_handshake.zig:831](../../src/crypto/tsumugi_handshake.zig#L831)). MeshPass token validation enforces realm, revocation epoch, expiry, and not-before via `issued_ms` ([src/proto/meshpass.zig:157](../../src/proto/meshpass.zig#L157), [src/proto/meshpass.zig:169](../../src/proto/meshpass.zig#L169), [src/proto/meshpass.zig:171](../../src/proto/meshpass.zig#L171), [src/proto/meshpass.zig:175](../../src/proto/meshpass.zig#L175)).

The secured adapter exposes `peerNodeKey`, the authenticated raw Ed25519 signing public key recovered from the Tsumugi session, and the daemon uses that key to verify cross-mesh oper grants ([src/daemon/secured_s2s_link.zig:127](../../src/daemon/secured_s2s_link.zig#L127), [src/daemon/server.zig:2650](../../src/daemon/server.zig#L2650)).

## Channel CRDT model

The implemented channel CRDT covers one channel. Membership is an add-wins observed-remove set keyed by stable `MemberId`, and modes are last-writer-wins registers ordered by `(HLC, replica_id)` ([src/substrate/suimyaku/channel_crdt.zig:4](../../src/substrate/suimyaku/channel_crdt.zig#L4), [src/substrate/suimyaku/channel_crdt.zig:6](../../src/substrate/suimyaku/channel_crdt.zig#L6)). HLC is a wall-clock timestamp plus logical counter, encoded to the LWW key in the same order as its compare function, so cross-host LWW uses wall-clock HLC ordering rather than local monotonic age checks ([src/substrate/suimyaku/clock.zig:16](../../src/substrate/suimyaku/clock.zig#L16), [src/substrate/suimyaku/clock.zig:21](../../src/substrate/suimyaku/clock.zig#L21), [src/substrate/suimyaku/clock.zig:77](../../src/substrate/suimyaku/clock.zig#L77), [src/substrate/suimyaku/clock.zig:86](../../src/substrate/suimyaku/clock.zig#L86)). Member status stores voice/op/owner/founder bits ([src/substrate/suimyaku/channel_crdt.zig:26](../../src/substrate/suimyaku/channel_crdt.zig#L26)).

| Entity | Current representation | Merge behavior |
| --- | --- | --- |
| Membership | Per-member add dots plus causal context (`MemberEntry.adds`, `MemberEntry.context`). | Local join increments the version vector, observes the new dot, and appends a `MemberAdd`; local part clears observed adds; merge removes adds covered by remote context and keeps concurrent live adds. |
| Member status | `MemberStatus` bitset stored on each live add. | `memberStatus` unions the status bits across live adds, so concurrent adds accumulate prefix bits. |
| Boolean channel modes | Goryu LWW registers for invite-only, moderated, no-external, topic-protected, secret. | Register merge decides by timestamp/replica ordering. |
| Parameter modes | LWW registers for key and limit. | Same LWW path as booleans, with bounded key length and `present` flags. |

The join mutator creates a new dot from the version vector, advances HLC, observes the dot in both local and delta contexts, removes covered adds while keeping the new dot, and returns a delta CRDT ([src/substrate/suimyaku/channel_crdt.zig:180](../../src/substrate/suimyaku/channel_crdt.zig#L180), [src/substrate/suimyaku/channel_crdt.zig:189](../../src/substrate/suimyaku/channel_crdt.zig#L189), [src/substrate/suimyaku/channel_crdt.zig:195](../../src/substrate/suimyaku/channel_crdt.zig#L195)). Merge combines HLC, version vectors, member adds/contexts, and mode registers ([src/substrate/suimyaku/channel_crdt.zig:229](../../src/substrate/suimyaku/channel_crdt.zig#L229), [src/substrate/suimyaku/channel_crdt.zig:245](../../src/substrate/suimyaku/channel_crdt.zig#L245)). The merge law is intentional: tests cover commutativity, idempotence, multi-order convergence, add-wins rejoin over observed part, and netsplit/heal convergence ([src/substrate/suimyaku/channel_crdt.zig:445](../../src/substrate/suimyaku/channel_crdt.zig#L445), [src/substrate/suimyaku/channel_crdt.zig:480](../../src/substrate/suimyaku/channel_crdt.zig#L480), [src/substrate/suimyaku/channel_crdt.zig:559](../../src/substrate/suimyaku/channel_crdt.zig#L559)).

## Burst, delta, and anti-entropy

`burst.zig` serializes a channel CRDT with magic `SBST`, version `1`, a header record, member delta records, and mode records ([src/substrate/suimyaku/burst.zig:39](../../src/substrate/suimyaku/burst.zig#L39), [src/substrate/suimyaku/burst.zig:45](../../src/substrate/suimyaku/burst.zig#L45)). The header carries HLC and the version vector ([src/substrate/suimyaku/burst.zig:103](../../src/substrate/suimyaku/burst.zig#L103)); member records use `delta_codec` membership views with context dots, live add dots, and derived remove dots ([src/substrate/suimyaku/burst.zig:130](../../src/substrate/suimyaku/burst.zig#L130), [src/substrate/suimyaku/burst.zig:147](../../src/substrate/suimyaku/burst.zig#L147), [src/substrate/suimyaku/burst.zig:214](../../src/substrate/suimyaku/burst.zig#L214)). Applying a burst builds an incoming CRDT and merges it into the target ([src/substrate/suimyaku/burst.zig:72](../../src/substrate/suimyaku/burst.zig#L72), [src/substrate/suimyaku/burst.zig:95](../../src/substrate/suimyaku/burst.zig#L95)).

`LinkSession` is the pure orchestration layer for initial burst, live deltas, gossip payloads, and Merkle-guided repair ([src/substrate/suimyaku/link_session.zig:1](../../src/substrate/suimyaku/link_session.zig#L1), [src/substrate/suimyaku/link_session.zig:3](../../src/substrate/suimyaku/link_session.zig#L3)). On establishment it observes the remote node in gossip and enqueues a burst ([src/substrate/suimyaku/link_session.zig:123](../../src/substrate/suimyaku/link_session.zig#L123)). On tick it emits gossip after `gossip_interval_ms` and repair summaries after `repair_interval_ms` ([src/substrate/suimyaku/link_session.zig:132](../../src/substrate/suimyaku/link_session.zig#L132), [src/substrate/suimyaku/link_session.zig:139](../../src/substrate/suimyaku/link_session.zig#L139), [src/substrate/suimyaku/link_session.zig:156](../../src/substrate/suimyaku/link_session.zig#L156)).

The repair module summarizes each member and present mode into Merkle entries, diffs local and remote summaries, builds key requests, builds record responses, and applies responses through a temporary CRDT merge ([src/substrate/suimyaku/anti_entropy_repair.zig:91](../../src/substrate/suimyaku/anti_entropy_repair.zig#L91), [src/substrate/suimyaku/anti_entropy_repair.zig:117](../../src/substrate/suimyaku/anti_entropy_repair.zig#L117), [src/substrate/suimyaku/anti_entropy_repair.zig:141](../../src/substrate/suimyaku/anti_entropy_repair.zig#L141), [src/substrate/suimyaku/anti_entropy_repair.zig:151](../../src/substrate/suimyaku/anti_entropy_repair.zig#L151), [src/substrate/suimyaku/anti_entropy_repair.zig:164](../../src/substrate/suimyaku/anti_entropy_repair.zig#L164)). Member repair prefers the canonical dense `DeltaView` payload and falls back to the compact version-vector frontier form only when dense causal-context expansion overflows; the compact encoder sorts context and adds so equal logical state hashes to identical bytes and avoids repair livelock ([src/substrate/suimyaku/anti_entropy_repair.zig:254](../../src/substrate/suimyaku/anti_entropy_repair.zig#L254), [src/substrate/suimyaku/anti_entropy_repair.zig:283](../../src/substrate/suimyaku/anti_entropy_repair.zig#L283), [src/substrate/suimyaku/member_compact.zig:88](../../src/substrate/suimyaku/member_compact.zig#L88), [src/substrate/suimyaku/member_compact.zig:313](../../src/substrate/suimyaku/member_compact.zig#L313)). `S2sPeer.repairRoot` exposes the current Merkle root for its state ([src/substrate/suimyaku/s2s_peer.zig:657](../../src/substrate/suimyaku/s2s_peer.zig#L657)).

Current-code boundary: `s2s_peer.zig` directly dispatches S2S `BURST`, `DELTA`, `GOSSIP`, and capability-gated repair frames. `LinkSession` still has its own pure repair-summary/request/response message-kind path over `peer_link` payloads, but the daemon's live S2S path uses the `S2sPeer` frame path through `S2sLink`/`SecuredLink` ([src/substrate/suimyaku/s2s_peer.zig:663](../../src/substrate/suimyaku/s2s_peer.zig#L663), [src/substrate/suimyaku/s2s_peer.zig:696](../../src/substrate/suimyaku/s2s_peer.zig#L696), [src/substrate/suimyaku/s2s_peer.zig:2118](../../src/substrate/suimyaku/s2s_peer.zig#L2118), [src/substrate/suimyaku/link_session.zig:254](../../src/substrate/suimyaku/link_session.zig#L254)).

## Gossip and membership view

`membership_view.zig` implements a bounded Suimyaku partial view: a small active view and larger passive view, with caller-supplied time and deterministic randomness ([src/substrate/suimyaku/membership_view.zig:1](../../src/substrate/suimyaku/membership_view.zig#L1), [src/substrate/suimyaku/membership_view.zig:40](../../src/substrate/suimyaku/membership_view.zig#L40)). Joining an overflowed active view demotes one active peer to passive ([src/substrate/suimyaku/membership_view.zig:169](../../src/substrate/suimyaku/membership_view.zig#L169), [src/substrate/suimyaku/membership_view.zig:195](../../src/substrate/suimyaku/membership_view.zig#L195)). Active failure removes the failed active member and may promote a passive candidate ([src/substrate/suimyaku/membership_view.zig:227](../../src/substrate/suimyaku/membership_view.zig#L227), [src/substrate/suimyaku/membership_view.zig:247](../../src/substrate/suimyaku/membership_view.zig#L247)).

## Route table and remote membership projection

The route table has two separate projections: nick-to-node routing and per-channel remote member rosters. Channel routing tracks node fanout counts, while `channel_members` stores remote `nick`, `node`, `status`, and `hlc` for NAMES/WHO-style projection ([src/substrate/suimyaku/route_table.zig:141](../../src/substrate/suimyaku/route_table.zig#L141), [src/substrate/suimyaku/route_table.zig:148](../../src/substrate/suimyaku/route_table.zig#L148), [src/substrate/suimyaku/route_table.zig:113](../../src/substrate/suimyaku/route_table.zig#L113)).

`MEMBERSHIP` frame payloads are compact binary records: `present`, `status`, `origin_node`, `hlc`, channel/nick length-strings, the member's propagated identity (username/realname/visible host), and an optional trailing `setter` ([src/proto/membership_event.zig:1](../../src/proto/membership_event.zig#L1), [src/proto/membership_event.zig:9](../../src/proto/membership_event.zig#L9)). `setter` is the nick that ran an explicit `/MODE` on the member, so a remote node renders the prefix change as `:<setter> MODE …` rather than the origin server; it is **omitted from the wire when empty**, keeping a join/part event byte-identical to the pre-setter format (the routing fields — `present`/`status`/`hlc` — drive convergence; identity and setter are display-only). The peer driver decodes inbound membership and applies it to the route table; malformed payloads are dropped without faulting the link ([src/substrate/suimyaku/s2s_peer.zig:350](../../src/substrate/suimyaku/s2s_peer.zig#L350)). Route-table membership updates are LWW by HLC; stale updates are ignored, newer present updates replace status/node, and newer parts remove the entry and prune empty channels ([src/substrate/suimyaku/route_table.zig:275](../../src/substrate/suimyaku/route_table.zig#L275), [src/substrate/suimyaku/route_table.zig:291](../../src/substrate/suimyaku/route_table.zig#L291)).

Nickname collision handling is deterministic and non-destructive. `resolveIncomingNick` keeps the wire nick when uncontested, renames a losing remote member to its stable mesh UID when it collides with a local or higher-priority remote holder, and lets same-account duplicates converge by LWW instead of minting a UID; the route table and peer driver explicitly never signal a kill for a collision loser ([src/substrate/suimyaku/route_table.zig:56](../../src/substrate/suimyaku/route_table.zig#L56), [src/substrate/suimyaku/route_table.zig:463](../../src/substrate/suimyaku/route_table.zig#L463), [src/substrate/suimyaku/route_table.zig:479](../../src/substrate/suimyaku/route_table.zig#L479), [src/substrate/suimyaku/s2s_peer.zig:1314](../../src/substrate/suimyaku/s2s_peer.zig#L1314), [src/substrate/suimyaku/s2s_peer.zig:1386](../../src/substrate/suimyaku/s2s_peer.zig#L1386)). The fallback UID is canonical, fixed-width base-36 over node/counter state, and the collision path derives it from the owning node's short id plus contested nick ([src/substrate/suimyaku/nick_collision.zig:52](../../src/substrate/suimyaku/nick_collision.zig#L52), [src/substrate/suimyaku/uid_alloc.zig:4](../../src/substrate/suimyaku/uid_alloc.zig#L4), [src/substrate/suimyaku/uid_alloc.zig:33](../../src/substrate/suimyaku/uid_alloc.zig#L33)).

The daemon announces local membership changes to all established peer links. JOIN sends the member's current mode bits after local join handling, and PART sends a `present=false` event after removing the local membership ([src/daemon/server.zig:3991](../../src/daemon/server.zig#L3991), [src/daemon/server.zig:3994](../../src/daemon/server.zig#L3994), [src/daemon/server.zig:4042](../../src/daemon/server.zig#L4042), [src/daemon/server.zig:4045](../../src/daemon/server.zig#L4045)). The shared announcer sends over both secured and plaintext established links and flushes each link's outbound buffer ([src/daemon/server.zig:3640](../../src/daemon/server.zig#L3640), [src/daemon/server.zig:3644](../../src/daemon/server.zig#L3644)).

On link-up, the daemon sends a one-time full local channel-member roster to that peer so the peer does not have to wait for future joins to build a roster ([src/daemon/server.zig:2663](../../src/daemon/server.zig#L2663), [src/daemon/server.zig:2699](../../src/daemon/server.zig#L2699), [src/daemon/server.zig:3611](../../src/daemon/server.zig#L3611)). The burst walks all local channels and local client members, sends their status bits, then flushes the peer link ([src/daemon/server.zig:3614](../../src/daemon/server.zig#L3614), [src/daemon/server.zig:3620](../../src/daemon/server.zig#L3620), [src/daemon/server.zig:3631](../../src/daemon/server.zig#L3631)).

## Link preservation across a hot upgrade (zero-drop USR2)

A Helix USR2 hot-upgrade historically dropped and re-dialed the secured mesh link (the accepted socket was closed at `execve`), so every deploy produced a visible netsplit — the peer's users QUIT/rejoined and derived-oper `+Y` prefixes re-announced. An **established** secured link is now *preserved* across the swap so the peer never sees a drop.

The mesh link stacks two layers, and only the outer one needs true crypto continuity:

- **Outer (Tsumugi record layer).** `sealSecuredLink` writes the live `Established` record keys + base nonces + peer identity, the per-direction record counters, the partial inbound-record carry (`rec_inbuf`), and all not-yet-sent sealed outbound into a new `.s2s_link` Helix capsule (`helix/s2s_snapshot.zig`), and the socket fd is preserved across `execve` (CLOEXEC cleared, as for carried TLS clients). The successor rebuilds the link established with `SecuredLink.resumeOuter` (no re-handshake) and continues the **exact** counter stream, so the untouched peer decrypts seamlessly. Counters are serialized verbatim and only ever advance, so no `(key, nonce)` pair is reused ([src/daemon/secured_s2s_link.zig](../../src/daemon/secured_s2s_link.zig), [src/crypto/tsumugi_handshake.zig](../../src/crypto/tsumugi_handshake.zig)).
- **Inner (CRDT roster).** The converged CRDT/route/registry state is intentionally *not* serialized. The successor stands the inner link up established from a small identity/caps header (`resumeEstablished` on `PeerLink`/`LinkSession`/`S2sPeer`/`S2sLink`), then sends a new `RESYNC` frame; the peer answers with the same full state burst a fresh establishment sends (`sendMeshStateBurstTo`). The roster/props/topics reconverge within one round-trip over the preserved encrypted stream. `RESYNC` is additive and the decoder skips unknown tags, so it is a safe rolling upgrade.

Fail-safe: a link with any seal/resume failure falls back to the old drop + re-dial for that one link — still no nonce reuse, just a one-time drop. A preserved link keeps its fd (the peer never re-dials) and its restored `s2s_connect_addr`/remote name let the `[mesh].connect` sweep recognize it, so no duplicate is dialed. Because the resumed link is *already* established, `adoptInheritedS2sLink` re-runs the peer-up registration (`publishServerLink`, `markPeerHealth`, `publishPeerCount`, `updatePartitionTransitions`) so LUSERS, MAP, the status page, and peer-health reflect the preserved peer immediately.

Source-verifiable boundary: the preserved-link path serializes record-layer keys/counters into a bounded outer resume state, resumes without re-running the AKE, and asks the peer for a full state burst after standing the inner link up established ([src/daemon/secured_s2s_link.zig:298](../../src/daemon/secured_s2s_link.zig#L298), [src/daemon/secured_s2s_link.zig:317](../../src/daemon/secured_s2s_link.zig#L317), [src/daemon/secured_s2s_link.zig:343](../../src/daemon/secured_s2s_link.zig#L343), [src/daemon/s2s_link.zig:176](../../src/daemon/s2s_link.zig#L176), [src/substrate/suimyaku/s2s_peer.zig:552](../../src/substrate/suimyaku/s2s_peer.zig#L552)).

`partition_detector.zig` is pure graph logic. ## Cross-server session reclaim

`session_reclaim_mesh.zig` defines portable reclaim tokens for session resume across the mesh. ## Current implementation boundaries

The source currently implements a frame-based peer driver and daemon relay path, not a full future banded protocol. The source today has Tsumugi-secured link establishment, CRDT burst/delta/gossip frames, route-table-scoped message relay, membership frame projection, nickname collision rename-to-UID handling, capability-gated repair frames, oper-grant sharing over secured links, and mesh-session reclaim. Where the code only provides pure helpers, this document calls that out: `writeNetjoin` exists, but the daemon does not call it; `LinkSession` contains a pure repair message path, while the live daemon peer path dispatches repair through `s2s_frame` messages in `S2sPeer`.
